API Documentation

Quick reference for using the REST API

Authentication

Include your API key in the request header:

X-API-Key: your-api-key-here

Available Endpoints

  • GET /api/v1/podcasts - List all podcasts
  • GET /api/v1/podcasts/{id} - Get specific podcast
  • GET /api/v1/podcasts/{id}/episodes - List episodes for podcast
  • GET /api/v1/episodes - List all episodes
  • GET /api/v1/episodes/{id} - Get episode with transcript
  • GET /api/v1/transcripts - List all transcripts
  • GET /api/v1/transcripts/{id} - Get specific transcript
  • GET /api/v1/search?q=query - Search transcripts

Query Parameters

  • limit - Results per page (default: 50, max: 200)
  • offset - Pagination offset (default: 0)
  • podcast_id - Filter by podcast ID
  • category - Filter by category
  • date_from - Filter by date (ISO format)
  • date_to - Filter by date (ISO format)
  • include_diarization - Include speaker segments (default: true)

Example Request

curl -H "X-API-Key: your-key" \
  "http://localhost:8000/api/v1/episodes?limit=10&transcribed_only=true"
